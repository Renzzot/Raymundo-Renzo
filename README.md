## 🎨 Design Inspirations
 
The following references shaped the visual and functional direction of this portfolio:
 
- **[Josh Comeau's Portfolio](https://www.joshwcomeau.com/)** — A developer portfolio that balances strong technical content with warm, approachable design. Inspiration for using subtle interactivity and personality without sacrificing clarity.
 
- **[Brittany Chiang's Portfolio](https://brittanychiang.com/)** — Clean dark-themed layout with a clear information hierarchy. Reference for how to present projects and experience in a way that works for both recruiters and technical clients.

## 🧠 Creative Process Documentation
 
### Initial Direction
The starting point was defining the audience first (direct clients, non-technical) and working backwards to the design. The earliest concept leaned heavily technical — showcasing code snippets and frameworks prominently — but this was quickly discarded, as it would alienate the primary audience of business owners and founders.
 
### Alternatives Considered
- **A one-page scrolling layout** was evaluated but ruled out in favor of a multi-section structure that allows cleaner navigation for users who arrive with a specific intent (e.g., looking only for contact info or a specific project).
- **An English-only version** was the original plan, but the decision to go bilingual (Spanish/English) came from recognizing that a significant portion of the target market is Latin American and values communication in their native language.
- **Dark mode as default** was inspired by references like Brittany Chiang's portfolio, but ultimately a light default with optional dark mode was chosen to feel more accessible to non-tech clients.
 
### Design Evolution
The concept evolved from "show what I know" to "show what I can do for you." This shift impacted every design decision: project descriptions now lead with the client's problem and the business outcome, not the tech stack. The hero section was simplified to a clear value proposition rather than a list of technologies.
 
### Key Design Decisions
- **Bilingual content** is handled with a visible language toggle, ensuring neither audience feels the site is not meant for them.
- **Typography and whitespace** are prioritized over decoration, following Refactoring UI principles to communicate professionalism without relying on complex visual effects.
- **Call to action** (contact form or direct email link) is persistent across all sections, reducing friction for clients who are ready to reach out.
- **Mobile-first approach** was adopted given that Latin American users predominantly access content via mobile on initial discovery.
